Supported Hosting Providers

caution

This list is incomplete. Please open a PR or contact us on Discord if the information is incorrect, or you want to add a new hosting provider!

caution

These providers are not verified by the Geyser team, and the server providers below are reported as working by members of the community.

info

Using a "Free Host" may not give you the best experience at all. If you want better support, more freedom to control your server, and learn how to run one, PLEASE pay for a server.

info

The below information is for the plugin versions of Geyser unless otherwise specified

Built-in Geyser​

  • Apex Hosting ‐ Full auto installation. Go to the Customizations tab in the panel and select 'Enabled' under the 'Geyser Auto Installer' option. Then, restart and connect to your server with your Java IP and port. See Apex's article for more details.
  • Arth Hosting ‐ Geyser is installed and configured on all servers by default. You can disable it in the 'Manage Plugins' menu.
  • Aternos ‐ Install Geyser in the Aternos plugin list and connect to your server with your Java IP and port. See Aternos's article for more details. Do note: Aternos automatically installs the latest version of the plugin, but does not auto update it. To update Geyser on Aternos, simply reinstall the plugin (no need to uninstall the plugin/deleting the plugin files).
  • Cloud Nord ‐ Go to their Crossplay Server hosting section and order your server. Instructions on how to join are provided.
  • CreeperHost ‐ Has a toggle within the control panel to automatically enable Geyser. May not be enabled by default, so you may need to toggle it and restart the server.
  • Cubes Hosting ‐ Install Geyser using the plugin manager. Then restart the server and Geyser will run on an additional port - you can check it in the server console. Geyser-Standalone can be set up through a support ticket.
  • exaroton ‐ Install Geyser in the exaroton plugin list and connect to your server with your Java IP and port. See exaroton's article for more details.
  • GGServers ‐ Can be installed to any server from their auto plugin setups. No further configuration needed.
  • Lilypad ‐ Tick the 'Enable Bedrock crossplay?' option when changing your server software to automatically install and configure Geyser + Floodgate. For more details, navigate to lilypad.gg/bedrock.
  • MCServerHost ‐ Under the server configuration, select 'Crossplay' to automatically install Geyser and Floodgate. Join with the connection address. To update Geyser and Floodgate, replace the plugin jars in the server manager and restart the server.
  • Minefort ‐ On the server dashboard under 'Connect Support', make sure 'Allow Bedrock' is enabled. Then connect via play.minefort.com and join using /server [servername], or connect via [servername].minefort.com.
  • Minehut ‐ Connect via bedrock.minehut.com and do /join [servername], or connect directly via [servername].bedrock.minehut.gg.
  • OMGServ ‐ Select Geyser in the Install Menu, it will be automatically installed. You can enable floodgate in the server properties on the dashboard.
  • Physgun ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port.
  • PiglinHost ‐ Order from the cross-platform servers page. Alternatively, contact their support by creating a support ticket or via discord for installation and configuration help.
  • Play Hosting ‐ Tick the 'Enable Bedrock crossplay?' option when changing your server software to automatically install and configure Geyser + Floodgate. For more details, navigate to help.play.hosting/minecraft/crossplay.
  • Pufferfish Host ‐ Geyser is installed and configured on all servers by default.
  • Server.pro ‐ Different instructions for different plans: On VPS, select Geyser in the Service Type Menu, Floodgate can be enabled on the dashboard configuration. For manual installation: Using a Premium or Free plan, use the same port as your Java server for the Bedrock port in the configuration file and connect that; for the Free plan, also enable the clone-remote-port option. If you are using a PRO or a VPS plan, you can use any port, which can be opened on the Firewall tab.
  • Snakecraft Hosting ‐ Select 'Paper + Geyser with Floodgate' under the Jar Type at checkout to install the Geyser plugin. Players will Connect with the same IP and port as you would on Java.
  • SRKHOST ‐ You can enable Geyser on the version changer page. Geyser will run on the given port by the host.
  • VemoxHost ‐ Full automatic installation. Go to one of the panel options and select the option to enable GeyserMC. Then, restart and connect to your server using your Java IP and port.
  • Virtual Gladiators ‐ Find the plugin in the control panel under the 'VG Recommended' category and restart your server. IP and port are the same as Java.
  • WiseHosting ‐ Both auto and manual installation available. Find the GeyserMC presets from the Properties tab in the Game Panel. There, click Install and restart your server. See Crossplay FAQ for additional details.
  • ZapHosting ‐ Find the plugin in the control panel and restart your server. IP and port are the same as Java.
  • 365Hosts ‐ Go to their Minecraft: Crossplay hosting section and order your server. Instructions on how to join are provided.

Support for Geyser​

  • Akliz ‐ Offers a working port forwarding option.
  • Aquatis ‐ Need to open a support ticket to get a UDP port, then use that as the bedrock port.
  • BisectHosting ‐ You must have a plan with a dedicated IP. In Geyser's config, uncomment the bedrock address and set it to the public IP of your server (e.g. address: 51.79.129.18). Leave the port as 19132. Under the home tab, select 'Enable UDP Network' and restart the server. See Bisect's article for full instructions. If you still cannot connect after following these instructions, contact Bisect Support as they reportedly have UDP disabled on some nodes.
  • Birdflop ‐ Open an allocation in the networks tab. If you have a dedicated IP, you can open a ticket to request allocation of the default Bedrock port (19132). Update the config to use the allocated port and restart the server.
  • Bloom.Host ‐ See Bloom's documentation for setup instructions.
  • Clovux ‐ Make sure your remote address is 'auto', uncomment bedrock address, and change 0.0.0.0 to your Java server's IP.
  • Consulhosting ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port.
  • Craft-Hosting ‐ Set the Bedrock port to the Java server's port and connect with that port; note that this provider appears to only provide service in Russia.
  • DedicatedMC ‐ Check DedicatedMC's documentation for specific instructions.
  • ElfIDC ‐ When placing an order, choose Paper/Spigot + Geyser, which will automatically install the Geyser plugin. Connect with the Java server's IP and port. Note: This provider only provides service in China.
  • EnviroMC ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port.
  • ExtraVM ‐ Make sure your remote address is 'auto', uncomment bedrock address, and change 0.0.0.0 to your Java server's IP.
  • FadeHost ‐ Set the Bedrock port to the Java server's port and connect with that port. Alternatively, buy a dedicated IP address to support a different port.
  • FakaHeda ‐ Set the bedrock port in the config to the Java server's port or use one of the available ports allocated to your server, and connect with that port.
  • FalixNodes ‐ Open a port yourself on the network page in the game panel, then use that port in the bedrock section of the Geyser config.
  • Ferox Hosting ‐ Open a port yourself in their panel and check out the knowledgebase article for how to install and configure Geyser.
  • Fluctis Hosting ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port.
  • FREAKHOSTING ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port.
  • FreeMcServer.net ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port.
  • Fusion Hosting ‐ Get Geyser as a plugin. Use the same port as your Java server for the Bedrock port in your config (either by setting it yourself, or enabling “clone-remote-port”) and connect with the same IP and port as you would on Java or create a port in the Network tab on the panel & use this for Geyser.
  • GameHosting.it ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port.
  • GameProHost ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port.
  • GPortal ‐ You have to adjust the port according to your query port. Scheme: Query port: xxx65. The ports from xxx66 to xxx70 are available. For example, if your query port is 12365, then Geyser can only run under the port range 12366-12370. Furthermore, you have to change the 'Bedrock' 'address' in the Geyser config to your IP address. You can find it above your query port. Don't forget to delete the # in front of `address'.
  • Heavynode ‐ Open a port yourself in the networking section of the control panel. Port 19132 is only available with a dedicated IP (contact support), otherwise you will need to use a randomly assigned port. To resolve further connection issues for servers located in Canada and the UK, contact their support with the info found here.
  • Hicoria ‐ Use one of the available ports allocated for your server for the Bedrock port in your config and connect with that port.
  • Hosterfy ‐ Use the same port as your Java server for the Bedrock port in your config (either by setting it yourself, or enabling clone-remote-port) and connect with the same IP and port as you would on Java. You can also ask for a new port.
  • HostEZ ‐ Geyser plugin supported with self-install or installed on request with its own port.
  • Host Havoc ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port.
  • Hosting-Minecraft ‐ Create a new port and use that port in the config. If you have a plan from the 'Boost' category, you will need to open a support ticket to get a port with UDP protocol enabled.
  • HumbleServers ‐ Set the Bedrock port to the Java server's port, or to one of the two extra ports, and connect with that port. If the subdomain doesn't work, use your regular numbered IP address.
  • KeKsHost ‐ Use the default bedrock port (19132) and the server ip to connect on Bedrock edition. No changes to the Geyser config are needed. You can also ask for a new port and use it instead.
  • Kinetic Hosting ‐ Open a port in the Network and Ports page, and set that port as the bedrock port in the Geyser config. Alternatively, use the Java server's port by enabling the clone-remote-port option in the config. Connect with the same IP as on Java, and the port that you have chosen. For more info see the hosts support article.
  • MC-HOST24.de ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), uncomment bedrock address, and change 0.0.0.0 to your Java server's IP. Connect with the Java IP and port.
  • MCFORFREE.DE ‐ Create an extra port in the game panel, then change the port in the bedrock section to the newly created port. To connect on Bedrock edition, use the Java server's IP and the port you've created. It may take a few minutes for the port to become active.
  • MCPEhost.ru ‐ Create an additional port in the server settings and enable UDP protocol, then use that as the bedrock port in the config. Connect with the Java IP and that new port.
  • Meloncube ‐ Contact the support for a UDP port to use for Geyser. Set that port as the bedrock port, and connect with it.
  • MineStrator ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port.
  • Minecraft-Hosting.pro ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port.
  • Modrinth Servers ‐ Check Modrinth's documentation for specific instructions.
  • Netbela ‐ Install Geyser with the dedicated Plugin installer. Use the same port as your Java server in your config. Connect with the same address and port as your Java server.
  • NFOservers ‐ Uncomment and set bedrock address to the server IP in the Geyser config file. As an alternative, you can run Geyser standalone separately on an Unmanaged VDS.
  • Nitrado ‐ Get Geyser as a plugin. Add 4 to the Port of the Java server. See here for specific instructions.
  • Nodecraft ‐ Use the default server port and 0.0.0.0 or your server IP as the host address.
  • OrionNodes ‐ Open a port yourself from the network page in the game panel, use that port in the bedrock section of the Geyser config.
  • PaperNodes ‐ Enable clone-remote-port (or manually set the Bedrock port to the Java port), and connect with the Java IP and port. Alternatively, you can contact the host to request an additional port or a dedicated IP.
  • Pebblehost ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port. See this video for a PebbleHost-specific tutorial.
  • PlanetNode ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port.
  • PUBCS ‐ Set the Bedrock port to the Java server's port and connect with that port, or upgrade to a plan that includes dedicated IP address to support a different port.
  • RamShard ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port.
  • Redline Hosting ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port.
  • Revivenode ‐ Set bedrock port to the Java server's port (either by setting it yourself, or enabling clone-remote-port). You can also create a secondary port in the network tab and use that. To fix random Unable to Connect issues, see here.
  • ScalaCube ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), uncomment bedrock address, and change 0.0.0.0 to your Java server's IP. Connect with the Java IP and port.
  • Shockbyte ‐ The port will be automatically set after you download Geyser and restart the server. Connect with the same IP and port as on Java. For Geyser Standalone installation instructions, see here.
  • Skynode.pro ‐ Go to 'Server Details', add a new port, and set the bedrock port in your config to that port. Connect with it and the Java server's address.
  • Sparked Host ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port.
  • STIPE ‐ Set the Bedrock port to the Java server's port and connect with that port; note that this provider only provides service in Australia.
  • SyntexHosting ‐ Set the Bedrock port to the Java server's port and connect with that port, or request a (free) additional port.
  • The Minecraft Hosting ‐ Try using 19132 as the Bedrock port, if that does not work, use the same Java server's port for the bedrock port in your config and connect with that port and the Java server's IP.
  • TNAHosting ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port.
  • TurboHost ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port.
  • UltimateSRV ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port.
  • VexyHost ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port.
  • Volcano Hosting ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port.
  • VoltHosting ‐ Enable clone-remote-port (or manually set bedrock port to the Java port), and connect with the Java IP and port. Alternatively, request an additional port, or buy a dedicated IP address. If you continue to have issues, please contact VoltHosting support for further assistance.
  • Vultam ‐ Set the bedrock port to the Java server's port, or request an additional port. The additional port can be 19132 if you buy a dedicated IP.
  • Wasabi Hosting ‐ Install Geyser with the plugin installer. Use the same port as your Java server in the Geyser config. Connect with the same address and port as your Java server.
  • Wepwawet ‐ Add a new port in the Network tab. Use this new port as the bedrock port.
  • WinterNode ‐ Set the Bedrock port to the Java server's port and connect with that port. Alternatively, request an additional port, or buy a dedicated IP address.
  • WitherHosting ‐ Either use the Java server's port, or create a port in the ports manager on the panel and use that for Bedrock. Refer to the hosts support article for further info.

Does not support Geyser​

  • MyArena ‐ Does seem to be working, but the Java version is too old in order for Geyser to run properly.
  • NiCraft ‐ Does not provide Bedrock support, and are affected by a firewall configuration issue, which prohibits most players from joining.
  • Feather ‐ Their proxy network does not support UDP. Therefore, Geyser is not supported. As an alternative, set up a paper server with e.g. playit.gg instead of port forwarding.
